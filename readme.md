 # PaymentForm — Система управления платежами  
       2  
       3 Тестовое полностековое приложение для управления пользователями, кошельками и  
         транзакциями.  
       4  
       5 ## 🏗 Структура проекта  
       6  
       7 *   **/Back_PaymentForm**: Серверная часть на ASP.NET Core (Clean  
         Architecture).  
       8 *   **/Front_PaymentForm**: Клиентская часть на Vue 3 + Vite + Pinia.  
       9  
      10 ---  
      11  
      12 ## 🛠 Предварительные требования  
      13  
      14 Перед запуском убедитесь, что у вас установлены:  
      15  
      16 1.  **Node.js**: Версия `20.19.0` или `>=22.12.0` (проверить: `node -v`).  
      17 2.  **.NET SDK**: Версия `8.0` или `9.0` (проверить: `dotnet --version`).  
      18 3.  **PostgreSQL**: База данных должна быть запущена.  
      19 4.  **Entity Framework Core CLI**: Для выполнения миграций.  
      20     *   Установка: `dotnet tool install --global dotnet-ef`  
      21  
      22 ---  
      23  
      24 ## 🚀 Быстрый старт  
      25  
      26 ### 1. Настройка базы данных (PostgreSQL)  
      27  
      28 В файле `Back_PaymentForm/PaymentForm.Web/appsettings.json` указана строка  
         подключения:  
      29 ```json  
      30 "DefaultConnection": "Host=localhost; Port=5432; Database=payment_form;  
         Username=postgres; Password=ВАШ_ПАРОЛЬ;"  
      31 ```  
      32 *   **Важно:** Убедитесь, что пользователь `postgres` существует и у него  
         есть права на создание БД.  
      33 *   **Пароль:** Если у вашего пользователя PostgreSQL есть пароль, добавьте  
         его в строку `Password=...;`.  
      34  
      35 ### 2. Запуск Backend  
      36  
      37 Перейдите в директорию веб-проекта:  
      38 ```bash  
      39 cd Back_PaymentForm/PaymentForm.Web  
      40 ```  
      41  
      42 Примените миграции для создания структуры БД:  
      43 ```bash  
      44 dotnet ef database update --project ../PaymentForm.DataBase/  
      45 ```  
      46  
      47 Запустите сервер:  
      48 ```bash  
      49 dotnet run  
      50 ```  
      51 *   API будет доступно по адресу: `http://localhost:5000` (или аналогичном —  
         проверьте вывод в консоли).  
      52 *   Swagger (документация API): `http://localhost:XXXX/swagger` (порт будет  
         указан при запуске).  
      53  
      54 ### 3. Запуск Frontend  
      55  
      56 Откройте новое окно терминала и перейдите в папку фронтенда:  
      57 ```bash  
      58 cd Front_PaymentForm  
      59 ```  
      60  
      61 Установите зависимости:  
      62 ```bash  
      63 npm install  
      64 ```  
      65  
      66 Запустите проект в режиме разработки:  
      67 ```bash  
      68 npm run dev  
      69 ```  
      70 *   Приложение откроется по адресу: `http://localhost:5173`  
      71  
      72 ---  
      73  
      74 ## 💡 Важные детали реализации  
      75  
      76 ### Backend:  
      77 *   **Архитектура**: Проект разделен на слои (Core, DataBase, Infrastructure,  
         Validation, Web).  
      78 *   **Валидация**: Используется `FluentValidation`. Ошибки ввода  
         обрабатываются автоматически.  
      79 *   **CORS**: Настроен на прием запросов от `http://localhost:5173`. Если  
         порт фронтенда изменится, обновите настройки в `Program.cs`.  
      80  
      81 ### Frontend:  
      82 *   **State Management**: Используется `Pinia` (хранилища в `/src/pinies`).  
      83 *   **Стили**: Tailwind CSS для быстрой и адаптивной верстки.  
      84 *   **Компоненты**: В проекте реализована собственная библиотека  
         UI-компонентов в `src/components/myUI`.  
      85  
      86 ---  
      87  
      88 ## 🛠 Исправление возможных проблем  
      89  
      90 1.  **Ошибка миграций**: Если команда `dotnet ef` не найдена, попробуйте  
         переустановить тулзу или проверить переменные окружения.  
      91 2.  **Проблемы с БД**: Убедитесь, что служба PostgreSQL запущена и порт 5432  
         открыт.  
      92 3.  **Связь фронта и бэка**: Если запросы блокируются (CORS), убедитесь, что  
         в `Program.cs` разрешен адрес, на котором запущен ваш фронтенд.
